# 🤝 Contributing to Hacker-ish Style Preloader

First off, thank you for considering a contribution to the **Preloader** project! We value clean, efficient, and aesthetic code. By contributing, you help us refine the ultimate cyberpunk preloader.

---

## 🚀 How to Join the Operation

1. **Fork** the repository to your own account.
2. **Create a Feature Branch** to keep your work isolated:
```bash
   git checkout -b feat/your-feature-name
   # or for bugfixes
   git checkout -b fix/your-issue-name
```
3. **Commit your changes** with descriptive, imperative messages:
```bash
 git commit -m 'feat: add glitch effect to progress bar'
```
4. **Push** your branch to your fork:
```bash
 git push origin feat/your-feature-name
```
5. **Open a Pull Request** against the main branch.

---

## 🛠️ Tactical Guidelines
**To maintain the project’s** integrity, please adhere to these standards:
* **Language:** Use Vanilla JavaScript (ES6+). No heavy frameworks unless strictly necessary.
* **Styling:** Keep CSS modular. Use CSS Variables (--color-primary, etc.) for all theming to allow for rapid color palette swaps.
* **Performance:** Animations are the core of this project. Ensure they remain Optimized; avoid layout thrashing and heavy repaints.
* **Documentation:** If adding a new feature, update the relevant sections of the *README.md*.

---

## 🐛 Bug Reporting

**Found a system vulnerability** or a visual glitch? Help us patch it:
1. **Search** existing issues to ensure it hasn’t been reported.
2. **Open a new Issue** and include the following intelligence:
* **Title:** Clear and concise summary.
* **Reproduction:** Step-by-step instructions to trigger the bug.
* **Environment:** Browser version, OS, and screen resolution.
* **Evidence:** Screenshots or screen recordings (highly recommended for UI/UX bugs).
